**Kubernetes Architecture has 2 Types of Nodes:**
1. Master Node
2. Worker Node


<b> Master Node Server  have 4 Types of Componets </b>  : 1) API Server
                                                                            2) Controller Manager (kube-controller-manager)
                                                                            3) Scheduler (kube-scheduler)
                                                                            4) ETCD

Client: * The Person who are managing the K8S
             * The Person who are Intracting the K8S are nothing but the Client

      1) API Server: It is a bridge interface between client to master components and 
                                components to components

      2) Controller Manager (kube-controller-manager): It is helps to recreation of POD’s
           Controller manager has 2 Types : 1) Repilication Controller manager (for master Node)
                                                                     2) NODE Controller manager (for Worker Node)

      3) Scheduler (kube-scheduler): Scheduler will Schedule right POD on right Worker Node


      4) ETCD : ETCD has how many POD’s you have, How many nodes you have, How many
                         Nodes you have. It is saved by Key=Values pair mode

     Worker Node Server  have 3 Types of Componets : 1) KUBELET
                                                                                             2) KUBEPROXY
                                                                                             3) POD’S
                                                                                 (By defaultly worker node have one docker pod)

1)	KUBELET: Work Node every information sends to API server . Like how many pods                
                       are there, CPU value, RAM value total end to end information shares

      2) KUBEPROXY: To Handle Network Information
    
      3) POD’s: POD is nothing but Group of Containers
