# Create a pod using manifest files
```
vim pod.yaml
```
```
apiVersion: v1
kind: Pod
metadata:
  name: mypod3
spec:
  containers:
    - name: cont1
      image: shammu101/cycle
      ports:
        - containerPort: 80
```
### apiVersion: v1
ğŸ‘‰ Tells Kubernetes which API version to use for creating this resource.

### kind: Pod
ğŸ‘‰ Specifies that we are creating a Pod resource.

### metadata:
ğŸ‘‰ Contains identifying information about the Pod.

### name: mypod3
ğŸ‘‰ Assigns the name mypod3 to the Pod.

### spec:
ğŸ‘‰ Defines the desired configuration (specification) of the Pod.

###  containers:
ğŸ‘‰ Lists the containers that will run inside this Pod.

### - name: cont1
ğŸ‘‰ Sets the name of the container as cont1.

###   image: shammu101/cycle
ğŸ‘‰ Specifies the Docker image to pull and run for this container.

###    ports:
ğŸ‘‰ Declares which ports the container will expose.

### - containerPort: 80
ğŸ‘‰ Exposes port 80 inside the container for application access.

# To access a Pod from the internet, we need to create a Service file.
## Services files are 3 Types
### 1) NodePort
ğŸ‘‰ NodePort to access from the internet.
### 2) ClusterIP
ğŸ‘‰ To provide the pod to pod communication.
### 3) LoadBalancer
ğŸ‘‰ LoadBalancer to distribute  equal to all workernodes.
## Create the Pod and Service files
```
vim pod.yaml
```
```
apiVersion: v1
kind: Pod
metadata:
  name: mypod3
  labels:
    app: dm
spec:
  containers:
    - name: cont1
      image: shammu101/dm
      ports:
        - containerPort: 80 
```

service.yaml
