# Create a pod using manifest files
```
vim pod.yaml
```
```
apiVersion: v1
kind: Pod
metadata:
  name: mypod3
spec:
  containers:
    - name: cont1
      image: shammu101/cycle
      ports:
        - containerPort: 80
```
### apiVersion: v1
ğŸ‘‰ Tells Kubernetes which API version to use for creating this resource.

### kind: Pod
ğŸ‘‰ Specifies that we are creating a Pod resource.

### metadata:
ğŸ‘‰ Contains identifying information about the Pod.

### name: mypod3
ğŸ‘‰ Assigns the name mypod3 to the Pod.

### spec:
ğŸ‘‰ Defines the desired configuration (specification) of the Pod.

###  containers:
ğŸ‘‰ Lists the containers that will run inside this Pod.

### - name: cont1
ğŸ‘‰ Sets the name of the container as cont1.

###   image: shammu101/cycle
ğŸ‘‰ Specifies the Docker image to pull and run for this container.

###    ports:
ğŸ‘‰ Declares which ports the container will expose.

### - containerPort: 80
ğŸ‘‰ Exposes port 80 inside the container for application access.

# To access a Pod, we need to create a Service file.

### 1) NodePort
ğŸ‘‰ NodePort exposes the application on a fixed port of each worker node, allowing external access using NodeIP:NodePort.
### 2) ClusterIP
ğŸ‘‰ ClusterIP exposes the application only within the Kubernetes cluster using an internal IP, and it cannot be accessed from outside.
### 3) LoadBalancer
ğŸ‘‰ LoadBalancer exposes the application to the internet by automatically creating a cloud providerâ€™s load balancer and assigning a public IP.
